Guia de Testes da API de Reservas (Localhost:8080)

Observação: Certifique-se de que a aplicação está rodando (./mvnw spring-boot:run) antes de iniciar.
1. Criar Nova Reserva (POST)

Endpoint: http://localhost:8080/api/reservas

Corpo (JSON):
JSON

{
  "nomeHospede": "Gabriella Siebert",
  "email": "gabriella@exemplo.com",
  "telefone": "62999999999",
  "dataEntrada": "2026-04-01",
  "dataSaida": "2026-04-10",
  "tipoQuarto": "SUITE",
  "status": "PENDENTE",
  "observacoes": "Quarto em andar alto, por favor."
}

    O que valida: Criação, validação de data futura e definição automática de dataCriacao.

2. Listar Todas as Reservas (GET)

Endpoint: http://localhost:8080/api/reservas

    O que valida: Retorno da lista completa do banco de dados.

3. Buscar por ID (GET)

Endpoint: http://localhost:8080/api/reservas/1

    O que valida: Recuperação de um registro específico ou erro 404 caso não exista.

4. Confirmar Reserva (PATCH)

Endpoint: http://localhost:8080/api/reservas/1/checkin (Nota: Use a ação de confirmar ou check-in conforme seu Controller)

    Ação Manual Recomendada: Primeiro use o endpoint de confirmação (se disponível) para mudar o status de PENDENTE para CONFIRMADA antes de tentar o check-in.

5. Realizar Check-in (PATCH)

Endpoint: http://localhost:8080/api/reservas/1/checkin

    Regra de Negócio: Só funciona se o status for CONFIRMADA e a data de entrada já tiver chegado.

6. Filtrar por Tipo de Quarto (GET)

Endpoint: http://localhost:8080/api/reservas/tipo_quarto/SUITE

    O que valida: O filtro implementado no ReservaRepository.

7. Buscar por Nome ou Email (GET)

Endpoint: http://localhost:8080/api/reservas/buscar?termo=Gabriella

    Importante: Este teste validará se você corrigiu o erro no método findBy...EmailContainingIgnoreCase que discutimos anteriormente.

8. Cancelar Reserva (PATCH)

Endpoint: http://localhost:8080/api/reservas/1/cancelar

    Regra de Negócio: Reservas com status EM_HOSPEDAGEM ou CONCLUIDA não podem ser canceladas.

9. Deletar Reserva (DELETE)

Endpoint: http://localhost:8080/api/reservas/1

    Regra de Negócio: Impede a exclusão se o hóspede já estiver no hotel (EM_HOSPEDAGEM).
